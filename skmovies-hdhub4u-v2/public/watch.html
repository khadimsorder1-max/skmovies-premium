How this works

Ad-free mode (default): we resolve the underlying .mp4 / .m3u8 / .mkv URL server-side, then load it in our own player at /player.html. The player uses Video.js + hls.js + Clappr, so it handles MP4 / HLS / MKV / WebM natively — no popups, no ads, no redirects.

Original mode: we wrap the original HDHub4u player URL inside a sandboxed iframe. The sandbox attribute strips away pop-ups, top-level navigation, and form submission — so the only thing the player can do is play video.

Resolved URLs are cached in Cloudflare KV + Cache API + localStorage for 24 hours — repeat plays don't burn Worker requests.
